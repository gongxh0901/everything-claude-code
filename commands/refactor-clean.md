# 重构清理

通过测试验证安全地识别和删除死代码：

1. 运行死代码分析工具：
   - knip：查找未使用的导出和文件
   - depcheck：查找未使用的依赖项
   - ts-prune：查找未使用的 TypeScript 导出

2. 在 .reports/dead-code-analysis.md 中生成全面报告

3. 按严重程度对发现进行分类：
   - SAFE：测试文件、未使用的工具
   - CAUTION：API 路由、组件
   - DANGER：配置文件、主入口点

4. 仅提出安全删除

5. 每次删除之前：
   - 运行完整测试套件
   - 验证测试通过
   - 应用更改
   - 重新运行测试
   - 如果测试失败则回滚

6. 显示已清理项目的摘要

在不先运行测试的情况下，绝不删除代码！
